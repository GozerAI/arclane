# Arclane Launch Guide

Step-by-step instructions to deploy Arclane to production.

---

## Phase 1: Accounts & Services

1. **VPS** — Get a Hetzner CPX21 (or similar: 3 vCPU, 4 GB RAM, ~€8/mo)
   - Ubuntu 22.04+
   - Note the server IP address

2. **DNS** — Point your domain at the server:
   - `A  arclane.cloud      → <SERVER_IP>`
   - `A  *.arclane.cloud    → <SERVER_IP>`
   - Allow 5–30 min for propagation

3. **Stripe** — Create a Stripe account at stripe.com
   - Create 3 products/prices: Starter ($49/mo), Pro ($99/mo), Enterprise ($199/mo)
   - Get the webhook signing secret (for `ARCLANE_WEBHOOK_SIGNING_SECRET`)
   - Set up a webhook endpoint: `https://arclane.cloud/api/businesses/{business_slug}/billing/webhook`
     - Events: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`

4. **Resend** — Sign up at resend.com
   - Verify `arclane.cloud` domain (add DNS records they provide)
   - Get the API key (for `ARCLANE_RESEND_API_KEY`)

5. **Sentry** (optional) — Create project at sentry.io
   - Platform: Python → FastAPI
   - Get the DSN (for `ARCLANE_SENTRY_DSN`)

---

## Phase 2: Server Setup

```bash
# SSH into your server
ssh root@<SERVER_IP>

# Download and run the setup script
curl -sL https://raw.githubusercontent.com/chrisarseno/arclane/main/deploy/setup-vps.sh | bash
```

This script will:
- Install Docker, git, ufw, fail2ban
- Configure firewall (SSH + 80 + 443 only)
- Create `arclane` system user
- Clone the repo to `/opt/arclane/repo`
- Copy `.env.example` → `.env` with a generated secret key
- Start services with `docker compose up -d --build`
- Run database migrations

---

## Phase 3: Configuration

Edit the environment file on the server:

```bash
nano /opt/arclane/repo/.env
```

Set these values:

```env
ARCLANE_ENV=production
ARCLANE_SECRET_KEY=<already generated by setup script>
ARCLANE_DATABASE_URL=sqlite+aiosqlite:///arclane.db

# Domain
ARCLANE_DOMAIN=arclane.cloud

# Upstream services
ARCLANE_CSUITE_BASE_URL=http://csuite:8007
ARCLANE_ZUULTIMATE_BASE_URL=http://zuultimate:8000
ARCLANE_VINZY_BASE_URL=http://vinzy:8001
ARCLANE_ZUUL_SERVICE_TOKEN=<generate with: openssl rand -hex 32>

# Stripe
ARCLANE_STRIPE_ENABLED=true
ARCLANE_WEBHOOK_SIGNING_SECRET=<from Stripe dashboard>

# Email
ARCLANE_RESEND_API_KEY=<from Resend dashboard>
ARCLANE_EMAIL_FROM_DOMAIN=arclane.cloud

# Monitoring (optional)
ARCLANE_SENTRY_DSN=<from Sentry>

# Scheduler
ARCLANE_NIGHTLY_HOUR=2
ARCLANE_NIGHTLY_MINUTE=0
```

After editing, restart:

```bash
cd /opt/arclane/repo
docker compose up -d
```

## Phase 4: Launch & Verify

```bash
# Check containers are running
docker compose ps

# Run migrations (if not already done)
docker compose exec arclane python -m alembic upgrade head

# Verify health
curl https://arclane.cloud/health
curl https://arclane.cloud/health/detailed

# Check logs
docker compose logs -f arclane
```

Expected health response:
```json
{"status": "ok"}
```

Detailed health (once C-Suite is connected):
```json
{"status": "ok", "checks": {"database": true, "csuite": true, "zuultimate": true}}
```

## Phase 5: Smoke Test

Run through the full flow manually:

1. **Landing page** — Visit `https://arclane.cloud/` — should show marketing page
2. **Sign up** — Create an account via the dashboard
3. **Create business** — Fill in the intake form, select a template
   - Should auto-trigger first cycle (uses bonus credit)
   - Should receive welcome email
4. **Dashboard** — Check metrics, content feed, credit gauge
5. **Live feed** — Visit `https://arclane.cloud/live` — should show SSE events
6. **Billing** — Test upgrade flow (use Stripe test mode first)
7. **Password reset** — Test forgot-password → reset-password flow

---

## Phase 6: Connect C-Suite

Arclane calls C-Suite for cycle execution. Ensure C-Suite is running and accessible:

```bash
# Test C-Suite reachability from the Arclane container
docker compose exec arclane curl -sf http://csuite:8007/health

# Test the cycle endpoint
docker compose exec arclane curl -X POST http://csuite:8007/api/v1/arclane/cycle \
  -H "Content-Type: application/json" \
  -d '{"business_slug": "test", "tasks": ["analyze_market"]}'
```

If C-Suite is on a different server, update `ARCLANE_CSUITE_BASE_URL` accordingly.

## Post-Launch Checklist

- [ ] SSL certificate auto-provisioned by Caddy (check `https://` works)
- [ ] Stripe webhook endpoint verified (send test event from Stripe dashboard)
- [ ] Resend emails delivering (check spam folder)
- [ ] Sentry receiving errors (trigger a test error)
- [ ] Nightly cycle runs at configured time (check logs next morning)
- [ ] Monthly credit reset works (verify on 1st of month)
- [ ] Backups configured (snapshot the VPS or backup the SQLite DB)

## Ongoing Maintenance

### Redeploy after code changes

```bash
ssh root@<SERVER_IP>
/opt/arclane/repo/deploy/redeploy.sh
```

This pulls latest, rebuilds, runs migrations, and verifies health.

### Monitor

```bash
# Live logs
docker compose logs -f arclane

# Container status
docker compose ps

# Health check
curl https://arclane.cloud/health/detailed
```

### Backup

```bash
# SQLite backup (run daily via cron)
cp /opt/arclane/repo/arclane.db /opt/arclane/backups/arclane-$(date +%F).db
```

### Rollback

```bash
cd /opt/arclane/repo
git log --oneline -5          # find the commit to roll back to
git checkout <commit-hash>
docker compose up -d --build
```
